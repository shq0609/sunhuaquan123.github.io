mysql基本操作

配置mysql开机自启动
1.chkconfig –add mysql
2.chkconfig mysqld on

1.service mysqld start/stop
2./etc/init.d/mysqld start/stop
命令关闭MySQL
mysqladmin -p -u root shutdown
检查mysql是否真正的启动
netstat -tulpn | grep 3306
查询进程
ps -ef | grep mysqld

连接mysql服务器

mysql -uroot -proot -h127.0.0.1 -P3306 -h IP服务器主机名IP地址 3306位默认端口

网络拓扑结构

网络的拓扑结构就是网络的各结点的连接形状和方法。

网络拓扑分类：

1.总线拓扑

总线型拓扑采用单根传输线作为传输介质（一般为同轴电缆），所有站点共享一条传输信道，一次只能由一个设备传输信号，目的站点匹配分组目的地址，复制内容。
优点：1.各站点互不影响。2.安装维护方便，电缆节约
结构简单。3.可靠性高，无源工作。
缺点：系统范围受到限制，需要中继器扩展，故障诊断困难，需要诊断各节点。故障隔离困难，总线故障，网络瘫痪。
2.星型拓扑
各结点通过单独线路连接中央节点，相互独立。中心节点一般为交换机。典型实例：专用交换机（private branch exchange，PBX）。
优点：介质访问控制简单，容易实现故障诊断和隔离
服务部署方便，网络拓展容易。
缺点：耗费电缆，安装维护成本高，过分依赖中心节点，各站点分布处理能力低。
3.环形拓扑
网络各节点通过一条通信线路连接，且数据以分组形式沿一定方向传输，形成闭环。 可以是单向或者双向。各站点有控制收发的访问逻辑。
优点：1.电缆长度短，类似总线型。2.扩展容易。3.单向传输，适合光纤，且路由控制简单。4.抗故障性能好。
缺点：单结点故障影响整个网络。检测故障困难。使用令牌传递方式，负载轻的时候，等待时间长。

4.树形拓扑

树形拓扑是分层结构，根接收信号，广播发送全网。
优点组网灵活，易于扩展，线路长度比星型短 故障隔离容易，隔离分支或节点。缺点：对根要求较高

5.网状拓扑
优点：不受瓶颈问题和失效问题的影响。可靠性高。
缺点：结构复杂，成本比较高；网状拓扑结构的网络协议比较复杂。

基于hexo搭建blog

hexo搭建个人博客

1.环境搭建

安装nodejs

ndoe -v npm -v 测试是否安装成功

安装hexo

npm install -g hexo-cli （-g表示全局安装hexo-cli脚手架）

验证是否安装成功 hexo -v

切换npm国内源
npm i nrm -g
nrm ls
nrm use taobao

安装git

git –version

git在安装后需要配置一下用户名和邮箱信息，在命令行工具中输入如下命令：
git config –global user.name “你的用户名”
git config –global user.email “你的邮箱”

gitm公钥配置见git文档

2.使用Hexo开始搭建博客

hexo init 项目目录下初始化
hexo n “文件名” 新建文件
hexo clean 清楚
hexo g 重新解析
hexo s 启动服务

3.将博客部署到github

登陆github创建仓库

创建仓库名“账户名.github.io”

在本地目录中安装github插件

hexo项目目录下输入cmd
npm install -save hexo-deployer-git （save表示仅在本项目中安装）

修改hexo项目目录下配置文件_config.yml

_config.yml下修改
deploy:
type: ‘git’
repo: github仓库地址
branch: master

git需要配置公钥

4.更换Hexo博客默认主题

下载主题

从github clone主题文件到hexo项目theme目录下
git clone https：//xxx.git themes/文件夹名称

修改_config.yml

修改该文件下themes位更换的文件夹名称

更新并重新部署

hexo clean 清理缓存
hexo g 解析生成
hexo s 运行hexo服务预览博客
hexo d 部署github通过github链接
